Configuring Apicurio Registry storage in a PostgreSQL database

This section explains how to configure Java Persistence API-based storage for Apicurio Registry on OpenShift using a PostgreSQL database Operator. You can install Apicurio Registry in an existing database or create a new database, depending on your environment.

Prerequisites

  • You must have an OpenShift cluster with cluster administrator access.

  • You must have already installed Apicurio Registry. See [installing-registry-operatorhub].

  • You must have already installed a PostgreSQL database Operator. See [installing-postgresql-operatorhub].

    If you have deployed a PostgreSQL database in another way, you can still use your database with the Apicurio Registry Operator, but you must create the required database configuration manually. See your PostgreSQL documentation for details.
Procedure

  1. Create a PostgreSQL database for your Apicurio Registry storage. For example, click Installed Operators > PostgreSQL Operator by Dev4Ddevs.com > Create database > YAML.

  2. Edit the database name to registry and edit any other database settings as needed depending on your environment, for example:

    apiVersion: postgresql.dev4devs.com/v1alpha1
kind: Database
metadata:
  name: registry
  namespace: my-project
spec:
  databaseCpu: 30m
  databaseCpuLimit: 60m
  databaseMemoryLimit: 512Mi
  databaseMemoryRequest: 128Mi
  databaseName: example
  databaseNameKeyEnvVar: POSTGRESQL_DATABASE
  databasePassword: postgres
  databasePasswordKeyEnvVar: POSTGRESQL_PASSWORD
  databaseStorageRequest: 1Gi
  databaseUser: postgres
  databaseUserKeyEnvVar: POSTGRESQL_USER
  image: centos/postgresql-96-centos7
  size: 1

  3. Click Create Database, and wait until the database is created.

  4. Click Installed Operators > Apicurio Registry > ApicurioRegistry > Create ApicurioRegistry.

  5. Paste in the following custom resource definition, and edit the database url and credentials to suit your environment:

    apiVersion: apicur.io/v1alpha1
kind: ApicurioRegistry
metadata:
  name: example-apicurioregistry
spec:
  configuration:
    persistence: "jpa"
    dataSource:
      url: "jdbc:postgresql://SERVICE_NAME.NAMESPACE.svc:5432/DATABASE_NAME"
      # e.g. url: "jdbc:postgresql://acid-minimal-cluster.my-project.svc:5432/registry"
      userName: "postgres"
      password: "PASSWORD"
      # ^ Optional

  6. Click Create and wait for the Apicurio Registry route to be created on OpenShift.

  7. Click Networking > Route to access the new route for the Apicurio Registry web console. For example:

    http://example-apicurioregistry.my-project.my-domain-name.com/
